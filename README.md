# AWS Serverless API using AWS Lambda

## Overview

This project demonstrates how to build a simple serverless REST API using AWS Lambda and Amazon API Gateway. Instead of managing traditional servers, the application uses AWS Lambda to execute backend logic only when requests are received, making the solution scalable, efficient, and cost-effective.

The project was developed as part of an AWS cloud learning program to understand the fundamentals of serverless computing and event-driven application development.

---

## Objectives

* Learn the fundamentals of serverless architecture.
* Create and deploy an AWS Lambda function.
* Integrate Lambda with Amazon API Gateway.
* Test API endpoints using HTTP requests.
* Understand how serverless services reduce infrastructure management.

---

## AWS Services Used

* AWS Lambda
* Amazon API Gateway
* AWS IAM
* Amazon CloudWatch

---

## Features

* Serverless REST API
* Event-driven execution
* Automatic scaling
* No server management
* CloudWatch logging for monitoring and debugging

---

## Project Workflow

1. Create an AWS Lambda function.
2. Write the backend code.
3. Configure an API Gateway endpoint.
4. Connect API Gateway with Lambda.
5. Deploy the API.
6. Test the endpoint using a browser or Postman.
7. Monitor execution logs using CloudWatch.
